Personalization and Authorship Guidance

This section is to help you present and personalize the assignment before submission:

- Write the `README.md` sections in first person to reflect what you did: e.g., "I implemented the POST, GET, PUT, DELETE endpoints, added pagination and search, and built a simple jQuery frontend." 
- Add screenshots under `/frontend/screenshots/` of the working UI to prove you ran it yourself.
- If the course requires a video demo, include a short screencast (2-3 minutes) where you open the UI and perform the CRUD actions.
- Add comments to key files (`/backend/routes/products.js`, `frontend/script.js`) summarizing why each route/function works.
- Run `bun install` and `bun run start` locally or `npm install` and `npm start` (if you prefer Node) and keep a small terminal log or screenshot of the server starting and the database connection working.
- Optionally, add a short `report.md` where you write about the design decisions you've made (e.g., use of prepared statements, error handling choices, minimal RESTful structure).

Remember: Academic integrity matters. If you used a sample or template, just mention that you used external references and adapt them.
